## 3. 잔여 리스크 등록부

### R-DESIGN-2026-03-25-01: 차트 런타임 색상 바인딩 가능성

**위험도:** LOW

**영향 범위:**
- completion-rate-chart.tsx 차트 렌더링
- Recharts 라이브러리 내부 색상 바인딩

**위험 설명:**
현재 `chartColors.ts`의 `CHART_HEX_COLORS`는 빌드 타임에 정의된 hex 값을 사용합니다. 다크모드 전환 시 Recharts가 CSS 변수를 직접 참조하지 못할 경우, 런타임에 색상이 변경되지 않을 수 있습니다.

**현재 완화책 (이미 구현됨):**
1. `getChartHex()` 함수로 CSS 변수 → hex 변환 브릿지 제공
2. SSR-safe한 `getComputedStyle()` 패백 포함
3. 다크모드 전환 시 Recharts의 `key` prop 강제 갱신으로 리렌더링 유도

**추가 완화책 (향후):**
1. Recharts 커스텀 테마 구성으로 CSS 변수 직접 참조 검토
2. 차트 렌더링 전 테마 감지 로직 강화

**영향 평가:**
- 현재 구현으로 라이트/다크 모드 전환 정상 동작 확인
- Recharts 버전 업데이트 시 회귀 가능성 있음

**책임자:** LMS 개발팀 알렉스
**모니터링:** 차트 색상 관련 사용자 리포트

---

### R-DESIGN-2026-03-25-02: 일부 브라우저 명암비 저하 가능성

**위험도:** LOW

**영향 범위:**
- 구형 브라우저 (Chrome < 111, Safari < 15.4)
- OKLCH 색상 공간 미지원 브라우저

**위험 설명:**
디자인 토큰은 OKLCH 색상 공간을 사용합니다. OKLCH를 지원하지 않는 구형 브라우저에서는 색상이 대체(fallback)되며, 이 과정에서 명암비가 의도보다 낮아질 수 있습니다.

**현재 완화책 (이미 구현됨):**
1. Tailwind 설정에서 `oklch` 미지원 시 자동 fallback
2. 다크모드 변수의 밝기(L) 값을 라이트 모드보다 0.12~0.15 높게 설정하여 안전성 확보
3. WCAG AA 기준(4.5:1)을 충족하는 색상 쌍 설계

**추가 완화책 (향후):**
1. 접근성 감사 도구로 정기 검증
2. 사용자 브라우저 통계 기반 fallback 색상 최적화

**영향 평가:**
- 주요 브라우저 최신 버전에서는 OKLCH 완전 지원
- 구형 브라우저 점유율 1% 미만 예상
- fallback 색상으로도 가독성 확보됨

**책임자:** 디자인팀 픽셀
**모니터링:** 접근성 감사, 브라우저 통계
