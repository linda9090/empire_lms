## 2. 잔여 리스크 등록부

### R-ISSUE9-2026-03-25-01: 초대 코드 브루트포스 시도

**위험도:** MEDIUM

**영향 범위:**
- `/api/invitations/[code]` 엔드포인트
- 무작위 코드 추적 시도

**위험 설명:**
6자리 nanoid는 62^6 ≈ 560억 조합으로 엔트로피 수준은 양호하나, 체계적인 브루트포스 공격에 대한 방어가 Rate Limiting에만 의존하고 있음. Rate Limiting 우회 시도가 있을 경우 유효한 코드 노출 위험이 존재함.

**발생 조건:**
- 공격자가 IP 풀을 활용해 Rate Limiting 우회 시도
- 자동화된 스크립트에 의한 대량 코드 검증 시도

**완화책:**
- **운영 중:** Rate Limiting 차단율 모니터링
- **운영 중:** 단일 IP에서 초당 5회 제한 강제 적용
- **향후:** 코드 추적 시도 시 CAPTCHA 도입 고려

**책임자:** 인프라보안팀
**완료기한:** 운영 지표 수집 후 2주간 리뷰
**우선순위:** P1 (운영 모니터링 필수)

**머지 후 추적 지표:**
```yaml
지표명: rate_limit_block_rate
측정: Rate Limiting에 의해 차단된 요청 비율
목표: < 0.1% (전체 요청 대비)
알림: 1% 초과 시 즉시 알림
데이터 소스: API 로그, HTTP 429 상태 코드
```

**머지 게이트:**
- [x] 본 리스크가 머지를 차단하지 않음을 합의
- [ ] 운영 지표 대시보드에 항목 추가
- [ ] 2주간 추적 결과 리뷰 예약

### R-ISSUE9-2026-03-25-02: 익명 검증 엔드포인트 남용

**위험도:** LOW

**영향 범위:**
- `GET /api/invitations/[code]` 엔드포인트
- 유효성 확인 시나리오

**위험 설명:**
초대 코드 수락 전 유효성 확인을 위해 인증 없이 접근 가능한 엔드포인트가 제공됨. 이로 인해 유효한 코드 여부 확인이 가능하며, 과도한 호출 시 DoS 공격의 표적이 될 수 있음. Rate Limiting이 적용되어 있으나, 정보 노출 우려가 남아있음.

**발생 조건:**
- 익명 사용자가 대량 코드 검증 시도
- 자동화된 도구에 의한 코드 유효성 스캔

**완화책:**
- **운영 중:** 비정상 검증 요청량 모니터링
- **운영 중:** 단일 IP/IP 대역에서의 검증 요청 패턴 분석
- **향후:** 검증 엔드포인트에 약한 인증(쿠키 기반) 도입 검토

**책임자:** 인프라보안팀
**완료기한:** 운영 지표 수집 후 2주간 리뷰
**우선순위:** P2 (향후 개선)

**머지 후 추적 지표:**
```yaml
지표명: abnormal_validation_request_rate
측정: 비정상 패턴의 검증 요청 비율
목표: < 0.5% (전체 검증 요청 대비)
알림: 2% 초과 시 즉시 알림
데이터 소스: API 로그, IP별 요청 패턴 분석
정의: 1시간 내 100회 이상 검증 요청을 보낸 IP
```

**머지 게이트:**
- [x] 본 리스크가 머지를 차단하지 않음을 합의
- [ ] 운영 지표 대시보드에 항목 추가
- [ ] 2주간 추적 결과 리뷰 예약

### R-ISSUE9-2026-03-25-03: 만료 경계 시각 오판

**위험도:** LOW

**영향 범위:**
- `POST /api/invitations/[code]/accept` 엔드포인트
- 만료 코드 수락 시나리오

**위험 설명:**
초대 코드의 만료 시각이 7일로 설정되어 있으나, 서버 시각과 클라이언트 시각의 차이, 또는 트랜잭션 경계 시점에서의 미세한 시차로 인해 만료 직전에 수락 시도가 410(Gone)으로 실패할 가능성이 존재함. 사용자 경험 저하 우려.

**발생 조건:**
- 만료 기한 1초 전에 수락 시도
- 서버 시간 동기화 지연 발생 시
- 데이터베이스 트랜잭션 지연 시

**완화책:**
- **운영 중:** 410/409 발생 비율 모니터링
- **향후:** 만료 시각 기준 여유 grace period(5분) 도입 검토

**책임자:** LMS 개발팀
**완료기한:** 운영 지표 수집 후 1달간 리뷰
**우선순위:** P2 (향후 개선)

**머지 후 추적 지표:**
```yaml
지표명: expiration_error_rate
측정: HTTP 410(만료) 및 409(중복) 상태 코드 발생 비율
목표: 410 < 0.1%, 409 < 0.05% (전체 수락 요청 대비)
알림: 목표 2배 초과 시 리뷰
데이터 소스: API 로그, HTTP 상태 코드 분석
```

**머지 게이트:**
- [x] 본 리스크가 머지를 차단하지 않음을 합의
- [ ] 운영 지표 대시보드에 항목 추가
- [ ] 1달간 추적 결과 리뷰 예약
